Test Scenarios:
1. Owner id is blank or null or not present
2. Owner id size is greater than 64
3. Owner contains char other than letters, digits, hyphens, and underscores
4. Owner is not present/created in system
5. date is null
6. date is not in expected date format
7. date should be either todays date or future date, not past date
8. (Success)No rules found for owner and day
9. (Success)at least 1 Rule found but no existing appointment/booking found
10. (Success)at least 1 Rule found and at least 1 existing appointment/booking found
There can be multiple scenarios in 10.




1. Owner id is blank or null or not present

{
    "success": false,
    "message": "Resource not found: availability/slots"

2. Owner id size is greater than 64
3. Owner contains char other than letters, digits, hyphens, and underscores
curl --location 'http://localhost:8080/api/v1/availability/$$123/slots?date=2025-08-09'
400 Bad Request
{
    "success": false,
    "message": "Validation failed: Owner Id can only contain letters, digits, hyphens, and underscores"
}

}
